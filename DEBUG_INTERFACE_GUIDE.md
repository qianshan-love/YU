## 🤖 Agent工作状态监控

### 功能说明：

1. **实时状态显示**
   - 6个Agent的当前状态
   - 状态指示灯：绿色=就绪，黄色=运行中，红色=错误
   - 自动刷新：每10秒更新一次状态

2. **Agent类型和颜色标识**
   - 任务规划 Agent - 蓝色 #667eea
   - 知识规范 Agent - 绿色 #27ae60
   - 撰稿生成 Agent - 黄色 #f39c12
   - 审校校验 Agent - 红色 #e74c3c
   - 版本控制 Agent - 紫色 #9b59b6
   - 成员管理 Agent - 深蓝 #3498db

3. **操作按钮**
   - **测试按钮** - 执行该Agent的测试任务
   - **详情按钮** - 查看Agent详细信息

### 状态说明：

| 状态 | 说明 | 指示灯 | 含义 |
|------|------|--------|------|
| ready | 就绪 | 绿色● | Agent已初始化，可以接受任务 |
| loading | 运行中 | 黄色闪烁● | Agent正在执行任务 |
| error | 错误 | 红色● | Agent出现异常 |
| unknown | 未知 | 灰色● | 状态未知 |

## 💬 大模型对话测试

### 功能说明：

1. **参数配置**
   - **系统提示词**：设置模型的角色和指导
   - **用户消息**：输入要发送给模型的内容
   - **温度参数**：控制生成随机性（0=确定，1=随机）
   - **最大Token数**：控制响应长度（100-4096）

2. **操作按钮**
   - **对话大模型**：发送消息到模型
   - **清空对话**：清空聊天历史

3. **对话显示**
   - 用户消息显示在右侧蓝色气泡
   - 模型回复显示在左侧灰色气泡
   - 支持HTML内容安全显示
   - 自动滚动到最新消息

### 使用示例：

#### 示例1：基础对话测试
```
系统提示词：你是县志编纂专家，擅长撰写符合规范的县志内容
用户消息：你好，请介绍一下自己
温度参数：0.7
点击：对话大模型
```

**预期结果：**
- ✅ 模型自我介绍
- ✅ 显示在对话区域
- ✅ 日志记录调用过程

#### 示例2：县志内容生成测试
```
系统提示词：你是县志编纂专家，熟悉《地方志书质量规定》
用户消息：请撰写县志概述章节的内容，包括地理环境、行政区划、人口民族等部分
温度参数：0.5
最大Token数：2048
点击：对话大模型
```

**预期结果：**
- ✅ 生成详细的县志概述内容
- ✅ 内容符合规范要求
- ✅ 显示token使用量

#### 示例3：多轮对话测试
```
第一轮：
用户消息：请问县志的概述章节应该包含哪些主要内容？
模型回复：（回答概述章节内容）

第二轮：
用户消息：这些内容应该如何组织结构？
模型回复：（回答结构安排建议）

第三轮：
用户消息：请根据以上建议，撰写一个概述章节的示例
模型回复：（生成具体内容示例）
```

## 💡 常见问题解答

### Q1：大模型调用无响应？
**A：**
1. 检查大模型服务是否正常运行
2. 查看网络连接是否正常
3. 查看系统日志中的错误信息
4. 尝试降低temperature和max_tokens参数
5. 检查API地址和模型名称是否正确

### Q2：Agent状态不更新？
**A：**
1. 等待10秒自动刷新
2. 点击"刷新状态"按钮手动刷新
3. 检查API端点是否正常响应
4. 查看浏览器控制台是否有JavaScript错误

### Q3：测试失败但不知道原因？
**A：**
1. 查看"系统日志"面板的详细错误信息
2. 查看"执行结果"面板的错误详情
3. 确认对应的Agent状态是否正常
4. 检查参数格式是否正确（JSON格式）

### Q4：如何查看更详细的日志？
**A：**
1. 在浏览器中打开开发者工具（F12）
2. 切换到Console标签查看JavaScript日志
3. 切换到Network标签查看API请求和响应
4. 检查每个API调用的状态码和返回内容

## 📈 性能监控建议

### 响应时间监控：

在"系统日志"中观察API调用时间：
- **大模型调用**：应<5秒（简单问答）
- **Agent执行**：应<10秒（简单查询）
- **状态查询**：应<2秒（轻量查询）

### 性能优化建议：

1. **减少大模型Token使用**：优化提示词，减少不必要输出
2. **缓存常用结果**：对重复查询进行缓存
3. **异步处理**：确保所有IO操作都是异步的
4. **批量操作**：对多个相关操作进行批处理
